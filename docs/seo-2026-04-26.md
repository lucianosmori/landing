## What the GSC CSVs actually say

- **9 clicks / 79 impressions / 11.39% CTR / pos 4.08** on a single indexed page. CTR is healthy at that position; the bottleneck is **impressions volume**, not CTR.
- **Only 2 queries surfacing**: `luciano mori` (4 / 25, pos 3.76) and `mori cloud` (0 / 1, pos 7). Translation: Google sees the site only as a **branded** result. Nothing topical surfaces yet. The new `Person.knowsAbout` + `llms.txt` + page copy targeting "DevSecOps", "agentic AI", "Black Hat 2025" are the pivot — give it 4–8 weeks to start ranking on those.
- **Mobile is 60% of impressions and 67% of clicks** (45/6 vs desktop 33/3). Mobile fidelity is the priority surface.
- **Geo asymmetry**: Canada is the only converting country (9/9 clicks). Impressions land in 21 countries but no clicks outside CA. Schema + `llms.txt` should help non-CA viewers connect "Luciano Mori" specifically to AWS/DevSecOps.
- **"Search appearance" is empty** — zero rich-result eligibility. The `Person` JSON-LD shipped above is what unlocks knowledge-panel candidacy.
- **The "2 not indexed pages"** is almost certainly resolved by canonical + sitemap. Expect that count to drop to 0 after Google recrawls. If it doesn't, *URL Inspection* will name the URLs (often `?` query strings or literal `/index.html`).

## Service account guide (for automated GSC pulls)

Overkill for current volume — included because it's the right shape once you grow. ~10 minutes one-time.

1. **Create a Google Cloud project** at https://console.cloud.google.com (e.g. `lucianomori-site-ops`).
2. **Enable the Search Console API**: *APIs & Services → Library* → search "Search Console API" → **Enable**.
3. **Create the service account**: *APIs & Services → Credentials → Create Credentials → Service account*. Name `gsc-reader`, no project IAM role needed.
   - Click into the service account → *Keys → Add key → JSON*. Save the file.
   - Note the email — `gsc-reader@<project-id>.iam.gserviceaccount.com`.
4. **Grant GSC access**: GSC → property → *Settings → Users and permissions → Add user*. Email = the service account email. Permission **Restricted** for analytics, **Full** if you also want it to submit sitemaps.
5. **Use it**:
   - **GitHub Action**: store the JSON key as a repo secret `GSC_SA_KEY`; weekly Action runs Python (`google-api-python-client`) calling `webmasters.searchanalytics.query` with dimensions `["query","page","country","device"]`, writes a CSV to `data/gsc-YYYY-MM.csv`, commits it.
   - **Local script**: same script, run ad-hoc, paste the CSV here.

**Security**
- Add `*.json` and `service-account*.json` to `.gitignore`.
- Rotate the key annually.
- The service account only has read access to the one GSC property.
